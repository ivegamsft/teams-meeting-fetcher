## Key Decisions

- **Human-in-the-loop pattern chosen**: Teams meeting creation, bot installation, and transcript generation require human interaction—cannot be fully automated without complex Teams bot infrastructure
- **Jest as test framework**: Provides structure (describe/test blocks, phases) and rich console output
- **Native Node.js (no SDKs)**: Uses `child_process.execSync` for AWS CLI and `https` module for Graph API to minimize dependencies
- **Serial execution**: maxWorkers: 1 prevents concurrent tests from interfering with shared resources
- **Rich console output**: Box-drawing and emojis guide humans through interactive process
