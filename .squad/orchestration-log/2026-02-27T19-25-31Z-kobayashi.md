## Assignment

Design transcript fetching architecture — recommend approach for connecting meeting notifications to transcript retrieval given timing constraints and Graph API capabilities.

## Outcome: ✅ Completed

**Architecture Recommendation:**
- **Phase 1 (Recommended):** Scheduled poller (every 5-15 min) in admin app
  - Queries DynamoDB for meetings where `status = 'scheduled'`, `endTime < now - 5min`, `onlineMeetingId` set
  - Calls `meetingService.checkForTranscript()` for each
  - Handles rate limiting, timeout after 1 hour for no-transcript meetings
- **Phase 2 (Production):** Event-driven via `getAllTranscripts` subscription (fallback to poller)

**Key Insights:**
- Calendar subscriptions tell us meetings are scheduled, not that they've ended
- Meeting end must be inferred from `endTime < now` (poll-based)
- Transcripts available 1-5 min after meeting ends (up to 20 min for long meetings)
- Batch enrichment needed first: call `fetchDetails()` on all 1105 meetings to populate `onlineMeetingId`
